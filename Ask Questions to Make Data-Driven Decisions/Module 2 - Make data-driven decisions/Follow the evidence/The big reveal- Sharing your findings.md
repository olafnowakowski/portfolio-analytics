# Reports and Dashboards Notes

## Why Data Presentation Matters

Data is only useful if people can understand it. Data analysts need ways to organize and present data so it becomes meaningful information for stakeholders.

Two common data presentation tools are:

- **Reports**
- **Dashboards**

Both are useful for data visualization, but they serve different purposes.

---

## Reports

A **report** is a **static collection of data** that is shared with stakeholders on a regular basis.

### Main purpose
Reports are useful for giving a **snapshot of high-level historical data**.

### Example
A finance company may send a **monthly sales report**.

### Benefits of reports

- can be shared on a regular schedule, such as weekly or monthly
- organized and easy to reference
- quick to design
- easy to use if maintained regularly
- based on static data that has already been cleaned and sorted

### Drawbacks of reports

- require regular maintenance
- are usually less visually appealing
- do not update automatically
- do not show live or changing data

---

## Dashboards

A **dashboard** displays **live, incoming, or frequently updated data**.

### Main purpose
Dashboards are useful for monitoring data in a more dynamic and interactive way.

### Benefits of dashboards

- give teams quick access to current information
- allow users to interact with the data using filters
- provide long-term value when data must be checked often
- save time when stakeholders need repeated access to updated information
- are usually more visually appealing

### Drawbacks of dashboards

- take more time to design
- may be inefficient if not used often
- require maintenance if the base data breaks
- can overwhelm users with too much information

---

## Report vs Dashboard

| Tool | Type of Data | Best Use | Strength | Limitation |
|---|---|---|---|---|
| Report | Static, historical data | Periodic summaries | Easy to reference and distribute | Does not show live data |
| Dashboard | Live or dynamic data | Ongoing monitoring | Interactive and up-to-date | More complex to design and maintain |

---

## Choosing Between a Report and a Dashboard

A data analyst should choose the tool based on stakeholder needs.

### Example
If stakeholders want to know about social media performance:

- a **monthly report** could show the number of new followers
- a **dashboard** could show live engagement across multiple platforms

The right choice depends on whether they need a **snapshot** or **ongoing monitoring**.

---

## Example: Using a Spreadsheet for a Report

A spreadsheet might contain wholesale company order details such as:

- order date
- salesperson
- unit price
- revenue

This data is useful, but it can be hard to understand in raw form. A report makes the information easier to read and share.

### Stakeholder request
They want a quick look at **revenue by salesperson**.

### Solution
Use a **pivot table** and a graph to summarize the data.

---

## What Is a Pivot Table?

A **pivot table** is a data summarization tool used in data processing.

It can be used to:

- summarize data
- sort data
- reorganize data
- group data
- count values
- total values
- average values

It can also transform:

- columns into rows
- rows into columns

### Why pivot tables are useful
Pivot tables help turn detailed spreadsheet data into a clear summary that is easier for stakeholders to understand.

---

## Dashboards in Practice

Dashboards can be created in tools such as **Tableau**.

A dashboard may include:

- interactive graphs
- multiple views of the data
- filters for location
- filters for date range
- other controls to change what the user sees

### Why this is helpful
Stakeholders can explore the data themselves instead of waiting for a new report each time.

---

## Key Takeaways

- Reports and dashboards are both useful tools for presenting data.
- **Reports** are best for static, historical, and regularly scheduled summaries.
- **Dashboards** are best for live, interactive, and ongoing monitoring.
- Reports are easier and faster to create, but dashboards are more dynamic.
- Pivot tables help summarize spreadsheet data and make reports easier to understand.
- A data analyst must choose the tool that best fits the needs of stakeholders.

---

## Short Summary

Reports give stakeholders organized snapshots of historical data, while dashboards provide interactive views of live or changing data. Reports are simple and easy to share, but dashboards are better for continuous monitoring. Both tools help analysts communicate data clearly and support better decision-making.
