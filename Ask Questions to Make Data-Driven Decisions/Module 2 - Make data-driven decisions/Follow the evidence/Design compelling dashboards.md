# Dashboards Notes

## What Is a Dashboard?

A **dashboard** is a visual tool that monitors **live, incoming data**. It brings information from multiple datasets into one central place so users can track, analyze, and visualize data more efficiently.

Data analysts use dashboards to:

- monitor performance
- answer business questions
- identify trends and patterns
- support decision-making
- solve problems using live data

---

## Why Dashboards Are Useful

Dashboards help both **data analysts** and **stakeholders** by making data easier to access and understand.

### Main benefits of dashboards

#### 1. Centralization
Dashboards combine multiple datasets into one place.

**For data analysts:**
- share a single source of data with stakeholders

**For stakeholders:**
- view data, projects, goals, and processes in one comprehensive place

---

#### 2. Visualization
Dashboards display live and updated data visually.

**For data analysts:**
- show incoming data in real time

**For stakeholders:**
- notice changing trends and patterns more quickly

---

#### 3. Insightfulness
Dashboards pull useful information from different datasets.

**For data analysts:**
- connect relevant data sources

**For stakeholders:**
- understand the story behind the numbers and track goals more effectively

---

#### 4. Customization
Dashboards can be tailored for different users or needs.

**For data analysts:**
- create custom views for a person, project, or presentation

**For stakeholders:**
- drill down into specific areas of interest or concern

---

## Important Limitation

Dashboards update automatically **only if the data structure stays the same**.

If the data structure changes:

- the dashboard design must be updated first
- live updates may stop working until the dashboard is fixed

---

## Tableau and Dashboards

One of the most powerful dashboard tools is **Tableau**.

### Why Tableau is useful

- supports many data sources
- handles large datasets
- allows real-time data updates
- offers advanced analytics features
- helps explore trends and patterns deeply
- creates interactive and engaging visualizations

### Limitation of Tableau

- takes time to learn well

### Why it is still valuable

Although Tableau has a learning curve, it is widely used because its dashboards are interactive, attractive, and effective for exploring data.

# How to Create a Dashboard

## 1. Identify the Stakeholders

Start by figuring out:

- who needs to see the dashboard
- how they will use it
- what questions they need answered
- what data matters most to them

### Why this matters
A dashboard should be designed around stakeholder needs, not just around the data itself.

---

## 2. Design the Dashboard

Decide what information should appear and how it should be organized.

### Good design tips

- use a clear header
- add short descriptions for each visualization
- place the most important information at the top

### Why this matters
A dashboard should be easy to follow and quick to understand.

---

## 3. Create Mockups if Needed

A **mockup** is a simple draft or sketch of the dashboard.

### Why use a mockup?

- helps plan the layout
- makes it easier to test ideas
- allows changes before building the final version

This step is optional, but many analysts find it helpful.

---

## 4. Select the Visualizations

Choose the chart type based on the data story you want to tell.

### Examples

- **Line chart** or **bar graph**  
  Best for showing change over time

- **Pie chart** or **donut chart**  
  Best for showing how parts contribute to a whole

### Why this matters
The right visualization makes the data easier to understand.

---

## 5. Create Filters as Needed

**Filters** allow users to show only part of the data while hiding the rest.

### Why filters are useful

- help users focus on specific information
- make it easier to identify patterns
- allow multiple people to use the same dashboard in different ways
- keep the original data unchanged

### Example
A dashboard might let users filter by:

- date range
- region
- product category
- department

---

## Dashboard Analogy

A dashboard is like the dashboard in an airplane.

It helps the user:

- see where things stand
- track movement and changes
- focus on key information
- navigate toward goals

In the same way, a data dashboard helps stakeholders follow the path of a project through data.

---

## Key Takeaways

- A **dashboard** is a live, visual tool for monitoring and exploring data.
- Dashboards save time by combining multiple datasets in one place.
- They help analysts and stakeholders through:
  - **centralization**
  - **visualization**
  - **insightfulness**
  - **customization**
- Tableau is a powerful tool for building dashboards, especially for large and interactive datasets.
- To create a dashboard:
  1. identify stakeholders
  2. design the dashboard
  3. create mockups if needed
  4. choose the right visualizations
  5. add filters when useful
- Dashboards are most effective when they are clear, focused, and designed around user needs.
