# Dynamic Revenue Metrics and Amazon’s Online Sales Process

## Main Idea
This lesson compares **traditional enterprise sales** with **online retail sales**, using **Amazon.com** as an example of a company that uses dynamic revenue metrics extremely well.

Enterprise sales usually depend on personal one-on-one meetings between salespeople and customers. Online retail is the opposite: customers make purchases without interacting with a salesperson.

Amazon succeeds because it makes the online experience feel personalized, useful, and efficient, even though the process is handled by computer systems.

---

# Enterprise Sales vs Online Retail

## Enterprise Sales
Enterprise sales usually involve:

- personal contact
- phone calls
- in-person meetings
- salespeople building relationships with customers
- long sales cycles

These sales rely heavily on traditional revenue metrics.

## Online Retail
Online retailers try to convince customers to buy without direct human interaction.

Instead, they use:

- websites
- mobile apps
- search tools
- recommendations
- real-time customization

### Important lesson
A successful online retailer must make the digital experience feel helpful and personal, not robotic.

---

# Amazon as an Example

Amazon is used as an example because it does an excellent job of using **dynamic revenue metrics**.

Amazon studies user behavior to improve the shopping experience and increase revenue.

This includes studying:

- clicks
- cursor movements
- page-to-page movement
- search terms
- browsing behavior

This kind of data is called **clickstream data**.

---

# What Is Clickstream Data?

## Definition
**Clickstream data** is the pattern of a user’s activity on a website.

It may include:

- what the user clicks
- how the cursor moves
- which pages the user visits
- how the user moves from page to page
- what the user searches for
- what the user views or ignores

### Why it matters
Clickstream data helps companies understand what users are doing in real time and adjust the experience to improve the chance of a purchase.

---

# Why Amazon’s System Is Powerful

Amazon’s system combines two types of information:

1. **Pre-processed data**
2. **Real-time user activity**

## Pre-processed data
This includes information Amazon has already organized in advance, such as:

- product databases
- book records
- text indexes
- subject categories
- past user behavior
- similar customer behavior

## Real-time user activity
This includes what the current user is doing during the session, such as:

- search terms
- clicks
- page movements
- browsing choices

### Important lesson
Amazon combines historical data with real-time behavior to customize the user experience while the user is still on the site.

---

# Real-Time Customization

## Definition
**Real-time customization** means changing an individual user’s experience while their session is happening.

Amazon can customize what a user sees based on:

- that user’s past behavior
- current search terms
- current clickstream behavior
- what similar users did in the past

### Why this is important
This is a strong example of making a business process change “right now.”

The system responds immediately to user behavior to improve the chance of a sale.
